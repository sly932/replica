# searchKnowledge 测试报告

## 工具简介

`searchKnowledge` 是 agent 的知识库语义检索工具：把分身的文章（article）切块、向量化后存入库，按查询词做向量相似度召回相关 chunk，供 agent 回答专业问题时引用。

## 测试目的与范围

验证「灌库 → 检索 → 召回」整条真实链路：

- `ingestArticle` 能把一篇 Markdown 文档切块、上下文化、向量化并落库（产生 ≥1 个 chunk）。
- `searchKnowledge.execute` 能按自然语言查询召回语义相关的 chunk。
- 返回结构符合约定 `{ ok, code, data }`，且正文命中部署相关关键词。

属于**集成测试**（真连 Supabase + ofox embedding + ofox anthropic 上下文化），非 mock。

## 用例清单

| 用例 | 输入 | 预期 | 实际 | 通过 |
|------|------|------|------|------|
| 灌库 | `ingestArticle(replicaId, '灵犀部署手册', deployment.md, file://…)` | `chunkCount > 0` | `chunkCount=5`，耗时约 37s | ✓ |
| 语义召回 | `execute({ query: '部署流程' })` | `ok=true`、`data.count>0`、正文含「灰度/staging/回滚/发布/prod」 | `count=5`，正文含灰度/staging/回滚等关键词 | ✓ |
| 清理 | 删 test replica | cascade 删 articles/chunks | 已删除 | ✓ |

## 环境依赖

- **Supabase**：replicas / articles / chunks 表（pgvector）。
- **ofox embedding**（OpenAI 兼容端点）：chunk 向量化，1536 维。
- **ofox anthropic**：ingest 时为 chunk 生成上下文摘要。
- **本地代理**：测试文件顶部用 `EnvHttpProxyAgent` 读取 `HTTP(S)_PROXY`（本机 `http://127.0.0.1:7897`）。
- **mock 种子文件**：`/Users/sly/projects/replica/mock/seed/articles/chenhao01/deployment.md`。
