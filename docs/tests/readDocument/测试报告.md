# readDocument 测试报告

## 工具简介

`readDocument` 是 agent 的文档分页读取工具：按 `docId` 从指定偏移 `curIdx` 读取一段长度 `readLen` 的原文，返回 `nextIdx`/`hasMore`/`total`，供 agent 逐页读完长文档。

## 测试目的与范围

- **分页**：默认每页 2000 字，`nextIdx` 推进、切片衔接、读完后 `hasMore` 翻转为 false 且 `nextIdx` 到达 `total`。
- **上限保护**：`readLen` 超过 2000 被钳到 2000。
- **错误路径**：空 docId → `[BAD_INPUT]`；不存在 → `[NOT_FOUND]`；跨分身越权读 → `[FORBIDDEN]`。

集成测试，真连 Supabase + ofox（ingest 一篇 >3000 字文档）。

## 用例清单

| 用例 | 输入 | 预期 | 实际 | 通过 |
|------|------|------|------|------|
| 灌库 | `ingestArticle` 一篇 3202 字文档 | `chunkCount>0` | `chunkCount=8`，约 51s | ✓ |
| page1 | `{ docId }` | `total=3202`、`text.length=2000`、切片=content[0,2000]、`nextIdx=2000`、`hasMore=true` | 全部符合 | ✓ |
| page2 | `{ docId, curIdx=2000 }` | 切片衔接、两页拼接=前缀、`hasMore=false`、`nextIdx=3202` | `nextIdx=3202`、`hasMore=false` | ✓ |
| readLen 钳制 | `{ docId, curIdx=0, readLen=5000 }` | `text.length=2000` | =2000 | ✓ |
| 空 docId | `{ docId: '  ' }` | `[BAD_INPUT]` | `[BAD_INPUT] docId 不能为空` | ✓ |
| 不存在 | `{ docId: <随机uuid> }` | `[NOT_FOUND]` | `[NOT_FOUND] 文档不存在` | ✓ |
| 跨分身越权 | 读他人 replica 的 article | `[FORBIDDEN]` | `[FORBIDDEN] 该文档不属于当前分身` | ✓ |
