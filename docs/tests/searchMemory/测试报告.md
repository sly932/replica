# searchMemory 测试报告

## 工具简介

`searchMemory` 是 agent 的记忆检索工具：按查询词对分身的 memories（episodic/semantic）做向量相似度召回，且只返回 enabled 且未软删的记忆。

## 测试目的与范围

- **召回**：enabled 记忆能被语义召回，结果结构含 `id/kind/content/similarity`。
- **过滤**：disabled / deleted 记忆即使内容强匹配也不被召回。
- **空结果**：无任何记忆的 replica 检索 → `count=0`，正文「未检索到相关记忆」。
- **错误路径**：空 query → throw `[BAD_INPUT]`。

集成测试，真连 Supabase + ofox embedding。

## 用例清单

| 用例 | 输入 | 预期 | 实际 | 通过 |
|------|------|------|------|------|
| 1+2 召回与过滤 | 灌 4 条均含「灰度发布」（2 enabled / 1 disabled / 1 deleted），`query='灰度发布怎么做'` | `count>=2`，仅召回 2 条 enabled，disabled/deleted 不出现，结构完整 | `count=2`，命中 enabled1+enabled2，过滤 disabled/deleted | ✓ |
| 3 空结果 | 空 replica，`query='灰度发布'` | `ok=true`、`count=0`、正文「未检索到相关记忆」 | `count=0`，正文匹配 | ✓ |
| 4 错误路径 | `query='   '` | throw `[BAD_INPUT]` | 抛出 `[BAD_INPUT]` | ✓ |
