# saveInsight 测试报告

## 工具简介

`saveInsight` 是 agent 的「保存推理洞见」工具：把 agent 自己推断出的结论写入 knowledge_items（`source=reasoning`、`status=pending_review`、`answer=conclusion`、带向量），待主人审核。

## 测试目的与范围

- **happy path**：`execute({ question, conclusion })` 返回 `{ ok, code, data:{ id, status } }`；回读 `source=reasoning`、`status=pending_review`、`answer=conclusion`、`question` 原样、embedding 1536 维。
- **错误路径**：空 question / 空 conclusion → `[BAD_INPUT]`；replica 不存在（外键违约）→ `[NOT_FOUND]`。

集成测试，真连 Supabase + ofox embedding。每个 case 用独立 replicaId 隔离。

## 用例清单

| 用例 | 输入 | 预期 | 实际 | 通过 |
|------|------|------|------|------|
| 1 happy path | `{ question, conclusion }` | `status=pending_review`，回读 source=reasoning / answer=conclusion / embedding=1536 | 全部符合，embedDim=1536 | ✓ |
| 2 空 question | `{ question:'  ', conclusion:'有结论' }` | `[BAD_INPUT]` | `[BAD_INPUT] question 不能为空` | ✓ |
| 3 空 conclusion | `{ question:'有问题', conclusion:'' }` | `[BAD_INPUT]` | `[BAD_INPUT] conclusion 不能为空` | ✓ |
| 4 replica 不存在 | replicaId 不入库 | `[NOT_FOUND]`（外键违约） | `[NOT_FOUND] 分身不存在` | ✓ |

## 环境依赖

- **Supabase**：replicas / knowledge_items 表（外键约束）。
- **ofox embedding**：洞见向量化，1536 维。
- **本地代理**：`EnvHttpProxyAgent`。
