# readKnowledgeItem 测试报告

## 工具简介

`readKnowledgeItem` 是 agent 读取单条知识条目（knowledge_item）全文的工具：按 `itemId`（且限定当前分身）返回 `question/answer/status/source`。

## 测试目的与范围

- **读取回验**：能读到已存在条目的完整字段，返回结构 `{ ok, code, data }`。
- **隔离**：读不存在 id → `[NOT_FOUND]`；读他人 replica 的条目 → 同样 `[NOT_FOUND]`（按 replica 隔离）。

集成测试，真连 Supabase + ofox embedding（插入时 `embedOne` 生成 question 向量）。

## 用例清单

| 用例 | 输入 | 预期 | 实际 | 通过 |
|------|------|------|------|------|
| 1 读取回验 | `{ itemId: <已插入id> }` | `ok=true`、`code=OK`，question/answer/status=approved/source=human 一致，正文含 question+answer | 全部一致 | ✓ |
| 2 不存在 id | `{ itemId: <随机uuid> }` | `[NOT_FOUND]` | `[NOT_FOUND] 未找到知识条目` | ✓ |
| 3 跨 replica 读取 | 用另一 replicaId 的 tool 读同一 id | `[NOT_FOUND]`（隔离） | `[NOT_FOUND] 未找到知识条目` | ✓ |

## 环境依赖

- **Supabase**：replicas / knowledge_items 表。
- **ofox embedding**：插入种子数据时 `embedOne` 生成 question 向量（读取本身不调 embedding）。
- **本地代理**：`EnvHttpProxyAgent`。
