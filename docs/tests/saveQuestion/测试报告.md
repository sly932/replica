# saveQuestion 测试报告

## 工具简介

`saveQuestion` 是 agent 的「登记待答问题」工具：当 agent 答不上来时，把问题写入 knowledge_items（`source=human`、`status=pending_answer`、带 question 向量），留给主人后续回答。

## 测试目的与范围

- **正常路径**：`execute({ question })` 返回 `{ ok:true, code:'OK', data:{ id } }`；回查库 `source=human`、`status=pending_answer`、`embedding` 1536 维、replica 隔离正确。
- **错误路径**：空白 question → throw `[BAD_INPUT]`。

集成测试，真连 Supabase + ofox embedding。

## 用例清单

| 用例 | 输入 | 预期 | 实际 | 通过 |
|------|------|------|------|------|
| 1 正常路径 | `{ question: '…主从切换 RTO 目标…' }` | 返回 `{ok,OK,data.id}`；回查 source=human / status=pending_answer / embedding=1536 维 / replica_id 隔离 | 全部符合，embeddingLen=1536 | ✓ |
| 2 错误路径 | `{ question: '   ' }` | throw `[BAD_INPUT]` | `[BAD_INPUT] question 不能为空` | ✓ |
