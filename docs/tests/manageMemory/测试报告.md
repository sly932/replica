# manageMemory 测试报告

## 工具简介

`manageMemory` 是**仅主人可用**的记忆管理工具，支持 `add`/`list`/`update`/`delete` 四种操作，并对新增做近似重复检测（向量相似度去重）。

## 测试目的与范围

- **CRUD**：add 落库（enabled、kind=semantic、embedding 1536 维）→ list 命中 → update 改写 content 并重算向量 → delete 软删 → list 验软删不再出现。
- **去重**：add 近似重复内容 → 业务失败 `{ ok:false, code:'CONFLICT', conflicts:[…] }`（不 throw），且不写入新行。
- **权限/错误**：非主人 → `[FORBIDDEN]`；delete 不存在 id → `[NOT_FOUND]`；add 空 content → `[BAD_INPUT]`。

集成测试，真连 Supabase + ofox embedding。

## 用例清单

| 用例 | 输入 | 预期 | 实际 | 通过 |
|------|------|------|------|------|
| 1 add | `{ type:'add', content }` | `ok=true`，回库 kind=semantic / enabled / embedding=1536 | 符合 | ✓ |
| 2 list | `{ type:'list' }` | count=1 且含刚 add 的 id | count=1 | ✓ |
| 3 add 近似重复 | 近似同义内容 | `ok=false, code=CONFLICT`，conflicts 含原 id，库里仍 1 条 | CONFLICT，未写入新行 | ✓ |
| 4 update | `{ type:'update', id, content2 }` | content 改写、embedding 重算 1536 维 | 符合 | ✓ |
| 5 delete | `{ type:'delete', id }` | 软删 `deleted=true`（行仍在） | deleted=true | ✓ |
| 6 list 验软删 | `{ type:'list' }` | count=0，已删 id 不出现 | count=0 | ✓ |
| 7 FORBIDDEN | 非主人 `isOwner:false` 调 list | `[FORBIDDEN]` | `[FORBIDDEN] 仅主人可管理记忆` | ✓ |
| 8 NOT_FOUND | delete 随机 uuid | `[NOT_FOUND]` | `[NOT_FOUND] 记忆不存在或不属于该分身` | ✓ |
| 9 BAD_INPUT | add 空 content | `[BAD_INPUT]` | `[BAD_INPUT] add 需要 content` | ✓ |

- **Supabase**：replicas / memories 表（pgvector）。
- **ofox embedding**：add/update 时向量化与去重相似度计算，1536 维。
- **本地代理**：`EnvHttpProxyAgent`。
