# searchConversation 测试报告

## 工具简介

`searchConversation` 是 agent 的历史会话检索工具：对会话 summary 的向量做语义召回，支持 `after`/`before` 时间窗过滤，帮 agent 找回相关的过往对话。

## 测试目的与范围

- **语义召回**：按主题召回最相关会话，最相关者排第一。
- **时间窗过滤**：`after` 排除更早会话、`before` 排除更晚会话、越界时间窗空召回。
- **错误路径**：空 query / 非法 ISO 时间 → `[BAD_INPUT]`。

集成测试，真连 Supabase + ofox embedding。两条种子会话：A 旧（技术·部署，2026-01-10）、B 新（生活·爬山，2026-06-20）。

## 用例清单

| 用例 | 输入 | 预期 | 实际 | 通过 |
|------|------|------|------|------|
| case1 语义召回 | `query='部署发布相关的技术讨论'` | `count>0`，召回 convA 且排第一 | convA 排第一 | ✓ |
| case2 after 过滤 | `query=部署…, after=2026-03-01` | 结果不含旧的 convA | 仅含 convB | ✓ |
| case3 before 过滤 | `query=爬山…, before=2026-03-01` | 结果不含新的 convB | 仅含 convA | ✓ |
| case4 越界时间窗 | `query=部署…, after=2030-01-01` | `count=0`，「未检索到相关会话」 | `count=0` | ✓ |
| case5 空 query | `query='   '` | throw `[BAD_INPUT]` | `[BAD_INPUT] query 不能为空` | ✓ |
| case6 非法时间 | `query='部署', after='不是时间'` | throw `[BAD_INPUT]` | `[BAD_INPUT] after 不是合法 ISO 时间` | ✓ |
