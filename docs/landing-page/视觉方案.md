# Replica 落地页 · 视觉方案 v0.1

> 产品：**Replica · 员工数字分身平台**（toB Agent 赛道）
> 一句话定位：3 分钟创建你的数字分身，替你回答重复问题、答不出转真人、越用越聪明、越用越懂你；经验可沉淀、可交接、可流动。
> 参考：`references/matrix-build/`（见同目录 `视觉拆解.md`）。本方案**借鉴其气质，但做差异化**，避免变成克隆。

---

## 0. 现状与约定

- 现有前端：`frontend/`，Vite 8 + React 19 + TS。
- 团队约定：**UI 先走 ui-preview skill 出 HTML 设计稿** → 确认 → 再写真实代码。
- 因此本页推进路径见 §6，**第一步只做静态 HTML 视觉稿**，先把"字体/配色/布局/滚动效果"方向定下来。

---

## 1. 设计基调（一句话）

**「克制留白 + 衬线大标题 + 一束暖光」**——延续 matrix 的杂志式高级感（亮色、衬线、真实质感），但把 matrix 的"自然风景"换成更贴合"数字分身"的母题：**人与分身、记忆生长、知识流动**。情绪关键词：**可信、温度、沉淀**。

与 matrix 的刻意区别：
- matrix 近乎纯黑白灰；我们**保留一个品牌强调色**（见 §3），让 toB 也有记忆点。
- matrix 用海湾/山坡风景；我们用**「分身/记忆」隐喻的动态视觉**（见 §5），避免撞脸。

---

## 2. 字体系统（直接沿用 matrix 思路，全部免费可商用）

| 角色 | 选型 | 说明 |
|---|---|---|
| 大标题 Display | **Instrument Serif** + 中文 **Noto Serif SC**（思源宋体 Light/Regular） | 英文斜体衬线 + 中文细衬线混排，复刻那种"好看的标题"。关键英文词（如 *Replica*、*digital twin*）用 italic span 提气质 |
| 正文 / UI | **Inter** 或 **HarmonyOS Sans SC**（替代 matrix 付费的 ABC Oracle） | 中英都有、易读、免费 |
| 小标签 / 等宽 | **DM Mono** | 全大写 + 字间距 ~1px + 半透明墨色，做眼标(eyebrow)、编号、"代码态"标签 |

> 落地：Google Fonts 引 Instrument Serif / DM Mono / Inter；Noto Serif SC、HarmonyOS Sans 走 CDN 或本地子集化（中文记得**按字符子集化**，否则体积大）。

## 3. 配色

| 用途 | 值 | 备注 |
|---|---|---|
| 底色 | `#FFFFFF` / 备用暖白 `#FAF9F7` | 亮色为主 |
| 主文字 ink | `#111418` | 近黑墨色 |
| 弱化文字 | `rgba(17,20,24,0.46)` | 标签、辅助说明 |
| **品牌强调色（待定，建议二选一）** | 暖橘 `#E8633A` / 墨蓝 `#2F4BFF` | 用于关键 CTA、链接 hover、强调下划线、分身光点 |
| 主按钮 | 近黑 `#0E1014` 实心 + 白字，**圆角 999px 胶囊** | 沿用 matrix |

> 强调色先在视觉稿里两版都出，让 sly 选。toB 偏稳重 → 墨蓝；想要"温度/人感" → 暖橘。

## 4. 页面结构（自上而下）

1. **Hero**：衬线大标题（如「让团队的经验，长出一个**分身**。」）+ 一句副文 + 邮箱胶囊 + 主 CTA。背景用 §5 的动态视觉。
2. **痛点/价值**：一句话点破"重复问题吃掉专家时间"，用 DM Mono 眼标分段。
3. **它怎么工作（⭐ 用 Scrollytelling）**：左 sticky 目录 + 右分段滚动 + 编号 01–04：
   `01 创建分身 → 02 替你答重复问题 → 03 答不出转真人 → 04 越用越聪明（记忆沉淀）`
   完全复刻 matrix 的 scroll-spy（左标题随滚动高亮）。
4. **能力 / 集成 Bento**：卡片网格——多渠道接入（飞书/Slack/网页挂件）、记忆库、转真人、知识交接。可借鉴 matrix 的"浮动 logo 药丸"展示集成。
5. **差异化背书**：对比 QuestionBase / Viven / Lucius 的一句话定位（见 `docs/竞品调研/`），克制不浮夸。
6. **Footer CTA**：全宽动态背景 + 最终 CTA + 邮箱胶囊 + 链接。

---

## 5. 动态视觉（替代 matrix 的风景，做出自己的母题）

matrix 首尾用 canvas 飘云 + footer 循环视频。我们建议：

- **Hero**：一个"**记忆/分身生长**"的动态——例如缓慢漂浮的光点逐渐连成一个人形/网络（canvas 或轻量 WebGL），或一段抽象暖光 loop 视频。强调"经验在生长、在流动"。
- **Footer**：沿用 matrix 的「全宽循环静音视频」手法做背景（`muted loop autoplay playsinline` + poster 兜底），内容换成贴合品牌的暖光/抽象人形画面。
- 一期可先用**渐变 + CSS 动效**占位，不必上来就做 WebGL；视觉稿阶段用 placeholder。

> 技巧备忘（详见拆解文档）：canvas 动态背景比静态图有生命力、比视频轻；scroll-spy 用 `IntersectionObserver`；胶囊按钮 `border-radius:999px`；眼标用 DM Mono 全大写 + letter-spacing。

---

## 6. 技术落地建议（回答"用什么搭"）

现在"先做页面、后接产品"，推荐**两步走**，既快又不浪费：

1. **第一步（现在）**：纯 **HTML + CSS +少量 JS** 单文件视觉稿，走 **ui-preview skill**，产物落 `docs/ui-preview/`。
   - 目的：最快确认字体/配色/布局/滚动效果方向，scroll-spy 手写即可，无框架负担。
2. **第二步（方向定了之后）**：移植进现有 `frontend/`（Vite + React 19）作为一个路由页 `/`（或独立 landing 入口）。组件化复用，后续好接产品数据。

> 不建议现在单独起 Next.js：现有前端已是 Vite+React，landing 并入同一工程最省事；除非后面明确要 SSR/SEO/多页营销站，再考虑迁移。
