# Replica 路演 PPT 大纲（v1）

> 约束（见 `参赛说明图片总结.md`）：**≤ 7 页**、官方模板、不嵌视频用截图、**4 分钟讲 + 2 分钟答辩**。
> 叙事重心：**场景创新性 30% + 产品完成度 25%** 占一半分，PPT 一半篇幅压在这两块。
> 每页都要能回答评委的潜台词（图6）。产品内容依据 `docs/产品/产品定义与执行蓝图.md`。

---

## 设计原则

1. **一页一个核心信息**，标题即结论（评委 4 分钟扫得完）。
2. **6 大痛点不平铺**——用一句话场景 + 一个硬核数据带过，避免流水账（数据来自 `research/`）。
3. **完成度靠"真截图 + 可上手"**讲，不靠口头承诺（呼应"开箱即用体验"环节）。
4. **反"换皮"**：差异化页明确打出「每人一个有人格分身 + 越用越懂你 + 可交接」组合拳。
5. 时间分配建议（4 分钟）：P2 痛点 40s → P3 方案 40s → P4 飞轮/Demo 90s → P5 技术 30s → P6 商业 30s → P7 收尾 10s。

## 7 页结构

### P1 · 封面 / 一句话定位
- 项目名 **Replica · 员工数字分身平台**
- Slogan：**3 分钟创建你的数字分身——替你回答重复问题、答不出转真人、越用越聪明、越用越懂你；经验可沉淀、可交接、可流动。**
- 副标题：toB Agent 赛道 · 单人作品 · 公网 Demo 链接
- 底部一行：星澜科技（虚拟公司）demo 场景

### P2 · 痛点（场景创新性 30% 的一半）
- 主标题结论：**"知识锁在少数人脑子里，人一忙就断档"**——六个侧面，同一个问题。
- 6 大真实场景，每条一句话 + 一个**已落地的硬数据**（完整出处见 `痛点数据与成本测算汇总.md`）：
  1. **专家被打断** → 每 2 分钟被打断一次、一天 275 次；恢复专注要 23 分钟（微软[1] / Gloria Mark[1]）
  2. **文档过时无人维护** → 企业 Wiki/SOP 半衰期不到 1 年，50% 知识不可检索（IDC[2]）
  3. **群机器人/客服无人盯 + 培训贵** → 人工答一次 $13.50 vs 自助 $1.84（差 7 倍）；客服年流失 30–45%（Gartner[3] / Insignia[3]）
  4. **新人 landing 无人带** → 新人 8–12 个月才完全产出，爬坡损失≈年营收 12%（Gallup/SHRM[4] / Docebo[4]）
  5. **离职/休假断档** → 42% 企业知识只在个人脑中；替换一名骨干=年薪 50%–200%（Panopto[5] / SHRM[5]）
  6. **跨组协作找不到人** → 每天 1.8h 在找信息/找对的人；每人每周 1.7h 重复答同一问题（McKinsey[6] / APQC[6]）
- 视觉：6 宫格场景图标 + 红色数字；页脚小字「参考来源：[1]…[6]…」（完整链接见 `痛点数据与成本测算汇总.md` 第六节）
- ⚠️ 这页要让评委"秒懂问题有价值"

### P3 · 解决方案 / 产品定义（场景创新性 + 完成度衔接）
- 主标题：**Replica = 每人一个会自进化、越用越懂你的数字分身**
- 五大能力（精简成图）：
  1. 3 分钟自助创建分身
  2. 双路自进化（答不出转真人沉淀 + 推理结论审批沉淀）
  3. 多层记忆（语义 + 情景，越用越懂你）
  4. 分身转移（经验可交接）
  5. 原生 IM（未来接飞书/钉钉）
- 一句话把 6 痛点 → 5 能力的映射收口

### P4 · 核心飞轮 + Demo 实拍（产品完成度 25% 主战场）
- 主标题：**一条真跑通的飞轮闭环（评委可自己上手）**
- 5 步飞轮 + 真截图（不嵌视频，用 Demo 页面截图）：
  1. 问命中（RAG + 人格化回答）
  2. 诚实 + 转人工（答不出不胡说，记为待回答）
  3. 真人补答 + 一键沉淀
  4. **再问命中**（最有冲击力的一幕）
  5. 越用越懂你（记忆召回）
  6.（可选）分身转移按钮
- 角标：公网 Demo 链接 / GitHub 链接 / 开箱即用

### P5 · 技术架构（技术深度 20%）
- 主标题：**RAG + Agent 工具设计 + 双路自进化 + 多层记忆链路**
- 架构图：Next.js 全栈 + Supabase(Postgres/pgvector/Storage) + Vercel
- 工程亮点：原文表/向量表分离、答不出边界、记忆抽取与召回、OpenAPI 批量灌库
- 一行体现"不是简单调 API"：分身 = 系统提示(人格+边界) + 工具集

### P6 · 商业潜力 + 差异化（商业潜力 15% + 反换皮）
- 上半：**为企业省多少钱**（取数见 `痛点数据与成本测算汇总.md`）
  - 一句话 ROI：**1,000 人企业，光"找人+找信息+重复答疑"每年隐性损耗约 $1,240 万；分身回收 40% = 每年近 $500 万**（还没算客服培训/新人爬坡/离职断档）
  - 三个买单人各一行：业务负责人（专家被打断/离职断档）、HR/L&D（新人爬坡，留存+52%）、IT/客服（培训贵+机器人乱答有 Air Canada 判例）
  - 宏观背书：Gallup 流动年损 $1 万亿 / Gartner 2026 对话式 AI 省 $800 亿客服成本
- 下半：差异化对比表（vs Viven / Lucius / Question Base）
  - 立足点三连：每人一个有人格分身 / 多层记忆越用越懂你 / 分身可转移交接
  - 红线提醒：飞轮落点是"沉淀回个人分身越用越像本人"，不是"公司共享手册"

### P7 · Roadmap + 收尾
- Roadmap：组织级分身 → 关系记忆 → 用户画像更个性化 → Proactive 主动行为
- 一句话愿景收尾 + 项目链接（Demo / GitHub / 团队信息）
