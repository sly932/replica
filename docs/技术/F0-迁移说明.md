# F0 · Vite → Next.js 前端迁移说明

> 任务：把 Replica 前端从 Vite + React + TS 迁移到 Next.js（App Router），纯前端、不接后端。
> 源：`frontend/`（Vite，保留不动）→ 目标：`web/`（新建 Next.js）
> 日期：2026-06-27 · 环境：Node 22.19 · Next 16.2.9 · React 19.2

## 1. 跑通情况

- `cd web && npm run dev` 正常启动（Turbopack，Ready < 200ms），`http://localhost:3000` 返回 200。
- `npm run build` 生产构建通过：TypeScript 类型检查 ✓、首页 `/` 静态预渲染 ✓。
- 交互全部验证通过（puppeteer 实测）：
  - 6 个功能页切换（对话 / 知识来源 / 知识条目 / 记忆 / 个人资料 / 设置）✓
  - 日 / 夜主题切换（`<html data-theme>`）✓
  - 对话三栏联动：点人员自动切到有内容的 Tab 并定位首条会话（如点「林晚」自动切到「我问 TA」）✓
  - 发消息：输入后回车 / 点发送，消息追加到当前会话气泡列表 ✓

## 2. web/ 关键结构

```

│   ├── page.tsx          渲染 <Workbench/>（Server Component）

│   ├── ui.tsx            Toggle 开关
│   ├── chat/ChatPage.tsx 对话三栏（人员 / 会话 / 聊天窗，含发消息）
```

路径别名：`@/*` → 项目根（`tsconfig.json` 默认）。组件统一用 `@/lib/...`、`@/components/...` 引用。

## 3. Next App Router 适配要点

- **client component 边界**：原 `App.tsx`（useState/useEffect）抽成 `components/Workbench.tsx` 并在文件顶加 `"use client"`。其余用到 hook 的组件（ChatPage、ui.tsx 等）由 Workbench 间接 import，自动归入 client 包，无需各自加指令。`app/page.tsx` 保持 Server Component，仅渲染 `<Workbench/>`。
- **全局 CSS**：三个 CSS 在 `app/layout.tsx` 里 import（App Router 要求全局样式在 layout 引入）。
- **去掉脚手架样板**：删除默认 `app/globals.css`（其 body margin/字体/Geist 字体变量会覆盖 base.css）、`app/page.module.css`、默认 page.tsx 的 Next logo 示例；layout 去掉 `next/font` 与默认 metadata。base.css 自带 `*{margin:0}` reset 与 body 样式，故不需要 globals.css。
- **入口差异**：Vite 的 `index.html` + `main.tsx`（createRoot）由 Next 的 `app/layout.tsx` + `app/page.tsx` 取代，无需手写挂载。

## 4. 踩的坑

- **相对 import 残留**：`data/mock.ts` 原来 `import ... from '../types'`，迁到 `lib/mock.ts` 后该相对路径失效。Turbopack **dev 模式不报**（照常渲染），但 `npm run build` 的 TS 检查会 fail：`Cannot find module '../types'`。已改为 `@/lib/types`。教训：迁移后务必跑一次 `npm run build` 兜底，dev 跑通 ≠ 类型正确。
- **create-next-app 模板自带 AGENTS.md/CLAUDE.md**：Next 16 脚手架生成了带「这不是你熟悉的 Next」警告的 AGENTS.md/CLAUDE.md（指向 `node_modules/next/dist/docs/`），属脚手架噪声，已删除。实测本项目用法（全局 CSS、`"use client"`、静态首页）与常规 App Router 一致，未触及 Next 16 的破坏性变更（如 `params` 改 Promise，本项目无动态路由）。

## 5. 视觉差异

无。深蓝 / 中性磨砂玻璃、标题栏红黄绿点、功能 bar、对话三栏、handoff 气泡、来源引用、日夜双模式均与原 Vite 版完全一致（CSS 原样照搬，组件 JSX 未改）。唯一额外元素是左下角 Next.js dev 工具的小圆 N 角标，仅 dev 模式出现，生产构建无。
