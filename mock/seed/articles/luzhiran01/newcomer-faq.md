# 新人高频问题 FAQ

本 FAQ 汇总星澜科技新人入职前 4 周最常问的问题，按「问题 → 找哪个组 / 谁」的形式整理。星澜全员远程办公，"找谁"默认指飞书私聊或拉群。**跨组找对人是 landing 的核心技能**，多用本表。

## 环境与研发

**Q：本地开发环境搭不起来，找谁？**
A：先看你所在组的工程 README / onboarding 文档（各组技术栈不同，没有统一脚本）。文档没覆盖、或报错卡住超过半天，找你的 mentor；没有 mentor 时找组长。各产品线技术负责人：灵犀（智能客服后端）找陈昊、磐石（BI）找苏晴、悟空（AI 编程助手）找顾承。

**Q：代码仓库 / 部署 / 数据库要权限，找谁？**
A：分情况：
- **统一登录 / SSO / 账号本身的问题**：找账号中台林晚（SSO 统一开通）。
- **某个具体仓库的 write / 部署权限**：找你所在组的组长或 mentor 走组内审批。
- **数据库 / BI 数据访问**：找磐石组苏晴。

**Q：第一个需求从哪开始？**
A：见《新人 Landing 全流程指南》——环境跑通后由 mentor / 组长派一个 good first issue。

## 模型与中台

**Q：想加大模型调用配额 / 申请模型权限，找谁？**
A：找 MaaS（模型即服务）中台赵启明，负责模型配额与调用权限。申请时说明用途、预估 QPS 和模型类型。

**Q：想接群机器人 / 消息推送，找谁？**
A：找推送中台周珉，负责群机器人与推送中台能力接入。

**Q：账号登录不上 / SSO 异常 / MFA 丢失，找谁？**
A：找账号中台林晚。

## 发布与稳定性

**Q：发布流程是什么？上线 / 回滚找谁？**
A：发布与监控由 DevOps 组石川负责。标准流程：组内 review 通过 → 合并 → 走发布流水线 → 观察监控。具体的流水线配置和发布窗口以 DevOps 的发布文档为准，首次发布建议拉上 mentor 陪跑。

**Q：线上告警 / 服务挂了，找谁？**
A：先看监控告警群里的值班信息，找 DevOps 石川。属于具体业务的问题再转对应产品线（灵犀陈昊 / 磐石苏晴 / 悟空顾承）。

## 行政与流程

**Q：请假怎么请？入离职、试用期、转正流程找谁？**
A：所有 HR 流程找人力资源温知夏，负责入离职与请假流程。请假在飞书审批里提单，按公司假勤制度走，紧急情况先口头同步组长再补单。

**Q：转正 / 首月反馈 / onboarding 课程相关，找谁？**
A：找 L&D 组路知然（luzhiran01）。新人公开课每双周三 15:00 飞书直播。

## 没人带 / 找不到对的人

**Q：我还没匹配到 mentor，组长也没安排人，问题没人答，怎么办？**
A：这是目前确实存在的痛点（详见《Mentor 制度与匹配规则》第四、五节）。最可靠的办法是飞书私聊 L&D 组路知然（luzhiran01），我们人工帮你协调对接。

**Q：我不知道这个问题该归哪个组，连"找谁"都不确定，怎么办？**
A：照本表对号入座通常能定位到人；实在判断不了，私聊路知然，我们帮你转介到对的组。
