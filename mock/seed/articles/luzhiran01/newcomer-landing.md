# 新人 Landing 全流程指南

欢迎加入星澜科技（Stellaris Tech）。本指南覆盖你入职首周到首月的关键动作清单。星澜全员远程办公，日常协作以飞书为主，本指南所有提到的"群"均指飞书群、所有"找谁"均指飞书私聊或拉群。建议入职第一天把本文档收藏到飞书云文档收藏夹。

## 入职第一天（Day 1）

入职当天 HR（温知夏）会通过飞书把你拉进「星澜新人报到」群，并触发以下流程：

1. **账号开通**：HR 在你报到当天提交账号申请，由账号中台（林晚）通过 SSO 统一开通。开通后你会收到一封激活邮件，覆盖飞书、代码仓库、内部 Wiki、MaaS 控制台等系统的统一登录。**首次登录务必绑定 MFA**。
2. **领取设备 / 配置远程环境**：远程办公的同学由 IT 寄送或自带设备，VPN 与堡垒机权限随 SSO 一并下发。
3. **加入组织架构**：确认自己在飞书通讯录里的部门、汇报线、花名是否正确，有误当天反馈 HR。
4. **认领 mentor**：理论上 Day 1 你会被分配一位 mentor（见《Mentor 制度与匹配规则》）。如果报到群里没人 @ 你说明 mentor 是谁，先别慌，第二天主动在群里问一句。

> 完成标志：当天能用 SSO 账号登录飞书和代码仓库。

## 首周清单（Day 1–5）

- [ ] **读完三份必读文档**：本指南、《Mentor 制度与匹配规则》、《新人高频问题 FAQ》。
- [ ] **搭好本地开发环境**：星澜没有统一的环境搭建脚本，每条产品线/中台的技术栈差异很大，**请以你所在组自己的工程 README / onboarding 文档为准**。例如灵犀（后端，陈昊）、磐石（BI，苏晴）、悟空（AI 编程助手，顾承）各有各的初始化流程。第一天找不到组内工程文档入口时，直接问你的 mentor 或组长要仓库地址。
- [ ] **申请研发权限**：代码仓库 read 权限随 SSO 默认开通，write / 部署 / 数据库等权限需要单独申请（见 FAQ「要权限找谁」）。
- [ ] **跑通第一个本地服务**：按组内文档把项目在本地跑起来，能访问到本地页面 / 接口即算通过。卡住超过半天就升级求助，不要自己闷头查。
- [ ] **参加新人 onboarding 直播**：L&D 组每双周三 15:00 在飞书办一场新人公开课（公司架构、研发流程、安全红线）。

## 第一个需求从哪开始

我们不建议新人第一周就接业务需求。标准路径是：

1. **环境跑通**后，由 mentor 或组长从需求池里挑一个 **good first issue**（通常是小 bug 修复或文档补全）给你练手。
2. 在飞书任务里认领该需求，按组内分支规范开发、自测、提 MR。
3. 走一遍完整的 **发布流程**（发布与监控相关问题见 FAQ，对应 DevOps 组石川）。
4. 第一个需求合并上线后，算作"landing 成功"的标志。

如果入职一周还没人给你派第一个需求，主动找组长要；组长也没响应时的兜底路径见《Mentor 制度与匹配规则》。

## 首月节奏（Week 1–4）

- **Week 1**：环境 + 必读文档 + 第一个 good first issue。
- **Week 2–3**：独立完成 1–2 个小需求，开始参加组内站会、需求评审。
- **Week 4**：与 mentor / 组长做一次转正前对齐，L&D 组会发一份《首月反馈问卷》，请如实填写——你的吐槽是我们改进 onboarding 的主要依据。

有任何流程层面（非技术）的疑问，飞书私聊 L&D 组路知然（luzhiran01）。
