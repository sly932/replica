# 模型接入与配额申请 SOP

> 本 SOP 是业务方接入 MaaS 推理服务、申请新模型与申请/调整配额的**唯一权威流程**。三条产品线（灵犀、悟空、磐石）的所有模型上线和扩容都必须走这里。如果你不知道「该找谁、走什么流程」——看本文。平台负责人 / 总审批人：赵启明（zhaoqiming01）。

## 一、什么时候需要走这个流程

- 业务方要上线一个**新模型**（新权重 / 新版本 / 新引擎）。
- 业务方现有配额不够，要**加 QPS / 加 TPM / 加 GPU 配额**。
- 要从逻辑隔离切到**物理隔离（独占部署）**。
- 要调整**降级路由链路**或灰度比例。

以上任何一项，**业务组自己改不了，必须经 MaaS 审批**。这是常见卡点：业务方常常不知道找谁，默认入口就是赵启明。

## 二、关键概念：配额三单位

| 单位 | 含义 | 典型申请粒度 |
|------|------|--------------|
| **QPS** | 每秒并发请求数 | 对话类按 5 / 10 / 20 档申请 |
| **TPM** | 每分钟 token 数（prompt + completion 合计） | 按 10万 / 50万 / 100万 档 |
| **GPU 配额** | 独占预留的显卡数 | 仅独占部署场景，按 1/2/4/8 卡 |

申请时**三者要一起报**：只报 QPS 不报 TPM，长输出场景会先撞 TPM 墙。

## 三、标准接入流程（以「灵犀要新增一个对话模型」为例）

1. **发起申请**：灵犀侧接口人（陈昊，chenhao01）在飞书发起《MaaS 模型接入申请》审批单，填写：
   - 模型名、参数规模、推理引擎倾向（vLLM / TRT-LLM）、上下文长度；
   - 预估流量：峰值 QPS、峰值 TPM、是否需要工具调用 / 流式；
   - 隔离要求：逻辑隔离（默认）还是物理隔离；
   - 降级预案：主模型挂了降级到哪个备用模型。
2. **MaaS 评审**：赵启明 + 推理后端简如评审可行性（引擎适配、卡型、显存占用）。**SLA：常规模型接入评审 2 个工作日内给结论。**
3. **资源排期**：通过后，SRE 乔宁排 GPU 资源、出部署方案。常规逻辑隔离模型，资源就绪 **T+3 工作日内**。
4. **灰度上线**：先按租户 10% 灰度，观测 TTFT / P99 / 错误率，稳定后放量。
5. **配额发放**：网关侧配置 QPS/TPM 配额，发 API Key，业务方联调验收。
6. **登记**：模型与配额登记入 MaaS 资源台账（赵启明维护）。

## 四、纯配额调整流程（不接新模型，只加量）

1. 业务接口人发起《MaaS 配额调整申请》审批单，写明：现有配额、目标配额、涨幅理由、预计生效日。
2. 赵启明审批；**单次 QPS 涨幅 ≤ 50% 且 TPM 涨幅 ≤ 50% 的常规扩容**，审批通过后 **T+1 工作日生效**。
3. 涉及加 GPU 配额（独占）的，需乔宁确认存量水位，可能需要采购/腾挪，时效另议。

## 六、紧急扩容 / 超大配额特批

常规流程覆盖「计划内、涨幅适中」的需求。但下列情况**不走上面的标准时效**：

- 线上突发流量、需要**小时级紧急扩容**；
- 单次 QPS 或 TPM **涨幅超过 50%** 的大幅扩容；
- 需要**抢占其他产品线 P0 资源**或临时挪卡；
- 一次性申请 **≥ 4 张 GPU** 的超大独占配额。

> **以上特批的具体审批路径、所需材料、谁来联签、几个工作日出结论——本 SOP 未固化（资源水位与优先级是动态的，按常规模板批容易误伤其他产品线）。遇到以上情况，请直接联系赵启明本人走特批，由他根据当时全局水位拍板。** 不要自行套用第四节的常规时效预估。
